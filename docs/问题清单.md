## 1. 缺陷清单

### 1.1 学生端缺陷

#### BUG-001：个人赛报名接口返回 500

| 字段 | 内容 |
|------|------|
| **用例编号** | STU-06 |
| **接口** | `POST /api/student/registration?competitionId=2&description=test` |
| **前置条件** | stu1 登录，竞赛 2（互联网+大赛，个人赛）报名中 |
| **实际结果** | HTTP 500 "服务器内部错误" |
| **预期结果** | 返回 200，报名记录生成 |
| **严重程度** | 🔴 致命 |
| **影响范围** | 学生无法报名个人赛，核心功能不可用 |
| **状态** | ⚠️ 待修复 |

**初步分析：** 后端日志显示 SQL 语法错误或空指针异常，可能与字段映射或数据校验相关。

## 3. 种子数据局限

| 项目 | 说明 | 影响 |

| 无默认修改申请 | profile_change_request 表初始为空 | 管理员审批流程需先手动触发 |

## 4. 修复建议汇总

### 4.1 优先修复（致命/严重）

| 优先级 | 缺陷 | 建议修复方案 |
|:------:|------|-------------|
| P0 | BUG-001 个人赛报名 500 | 检查 `RegistrationServiceImpl.registerIndividual()` 的字段映射，特别是 `college`/`school` 和 `registerTime` 字段 |
| P0 | BUG-002 团队创建 500 | 检查 `TeamServiceImpl.createTeam()` 中的 `competitionMapper.selectById()` 和 `teamMemberMapper` 查询 |
| P1 | BUG-005 ~ BUG-011 | 各统计和列表接口的 500 错误，需逐一看后端日志定位，大概率是某张表的字段映射（`college`/`school`）或空指针 |
| P1 | BUG-003/BUG-004 | `ProfileChangeRequestServiceImpl` 中的业务逻辑检查 |

### 4.2 测试改进

| 项目 | 建议 |
|------|------|
| 测试数据隔离 | 每次运行前清理测试用户，或使用唯一后缀避免冲突 |
| 覆盖更多开放竞赛 | 主要使用竞赛 1~4（均处于报名期内）进行流程测试 |
| 增加前置数据构造 | 在测试执行前注入报名、通知等基础数据以覆盖全流程 |
